SW-Requirements¶

SW Requirement Specification¶

This file contains the software requirements for the lobster-doxygen tool.

Software Requirements¶

General¶

sw_req_cli

Attribute Name

Attribute Value

Description

The software shall be a command-line interface (CLI) application.

Valid Status

valid

Note

N/A

Derived

N/A

Verification Criteria

Every feature shall be accessible via the command line. Therefore its verified by CLI tests.

sw_req_stdout_output

Attribute Name

Attribute Value

Description

The software shall show provide all user informational texts as console output on stdout.

Valid Status

valid

Note

N/A

Derived

N/A

Verification Criteria

Every feature user visible information shall be compared with a reference by capturing the console stdout output.

sw_req_stderr_output

Attribute Name

Attribute Value

Description

The software shall show provide all user error texts as console output on stderr.

Valid Status

valid

Note

N/A

Derived

N/A

Verification Criteria

Call the tool with an invalid option and capture stderr.

sw_req_rule_file

Attribute Name

Attribute Value

Description

The software shall abort with an error if requirements or justification are specified on file level.

Valid Status

valid

Note

Similar to the lobster-python behavior. File level is bad.

Derived

N/A

Verification Criteria

sw_req_rule_class

Attribute Name

Attribute Value

Description

The software shall abort with an error if requirements or justifications are specified on class/interface/namespace level and at least on of its methods.

Valid Status

valid

Note

Similar to the lobster-python behavior. Mixing levels is bad.

Derived

N/A

sw_req_unspecified

Attribute Name

Attribute Value

Description

The software shall output all supported levels without requirements or a justification too.

Valid Status

valid

Note

This is required to detect untraced sourcecode.

sw_req_no_trace

Attribute Name

Attribute Value

Description

The software shall output a warning and create a lobster common interchange file if doxygen XML contains no traces.

Valid Status

valid

Note

This is required to detect untraced sourcecode, but doesn’t break any toolchain flow.
